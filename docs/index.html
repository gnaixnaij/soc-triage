Host triage for incident response

Quickly collect system state information during compromise assessment. JSON output, watch mode, and automated collection for IR workflows.

What it collects

🖥️ System

  • Hostname and OS info
  • Logged-in users
  • Uptime, load, memory
  • Disk usage

    • 🌐 Network

  • Open ports / listeners
  • Active connections
  • ARP cache
  • Routing table

    • 🔐 Security

  • Running processes
  • Scheduled tasks / cron
  • Recent logins
  • Suspicious files

    • Quick start

    git clone git@github.com:gnaixnaij/soc-triage.git
cd soc-triage
python3 triage.py

    Usage

    # Quick triage (default output)
python3 triage.py

# Watch mode — collect every 30 seconds
python3 triage.py --watch --interval 30

# JSON output for ingestion
python3 triage.py --format json --output triage.json

# Include file hash scan
python3 triage.py --hash-files /tmp,/var/tmp

    When to use

    • Phishing alert — user ran an attachment
    • Ransomware investigation — collect before reboot
    • Lateral movement check — verify scope
    • Baseline comparison — compare with clean state
